PYTHON-LY MIDI PROGRAM FEATURE
==============================

Date: 2026-01-17
Affected package: python-ly (frescobaldi/python-ly)
Type: Feature enhancement (not a bug fix)

THE PROBLEM
-----------
When exporting LilyPond to MusicXML and importing into DAWs like Logic Pro,
no MIDI instruments are assigned. The user must manually assign instruments
to each track.

This happens because python-ly exports the instrument NAME but not the
General MIDI PROGRAM NUMBER.

Before (what python-ly exported):
    <midi-instrument id="P1-I1">
      <midi-channel>1</midi-channel>
    </midi-instrument>

After (what DAWs need):
    <midi-instrument id="P1-I1">
      <midi-channel>1</midi-channel>
      <midi-program>41</midi-program>      <-- ADDED
    </midi-instrument>

NOTE: <midi-name> was previously included but has been removed.
It is not a valid MusicXML 3.0 element and caused import errors in Dorico.

THE SOLUTION
------------
Two files were modified:

1. ly/musicxml/midi_sound_map.py
   - Added midi_program_map dictionary
   - Maps LilyPond instrument names to General MIDI program numbers (1-128)
   - Example: 'violin': 41, 'trumpet': 57, 'timpani': 48

2. ly/musicxml/create_musicxml.py
   - Import midi_program_map
   - In create_part(), add <midi-program> element when instrument is known

CODE CHANGES

File: ly/musicxml/midi_sound_map.py
Add after existing midi_sound_map:

    midi_program_map = {

File: ly/musicxml/create_musicxml.py
Change import:

In create_part(), after midi-channel, add:
    if midi in midi_program_map:
        midiprog = etree.SubElement(midiinstr, "midi-program")
        midiprog.text = str(midi_program_map[midi])

IMPORTANT: Scripts that use python-ly must add the project root to sys.path
BEFORE importing ly, otherwise the system-installed python-ly (without this
fix) will be used instead:
  _project_root = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
  sys.path.insert(0, _project_root)

This is because Python sets sys.path[0] to the script's directory (e.g.
scripts/), not the CWD, when running a script file.

EXPORT PIPELINE (ly_export.py)
------------------------------
Full pipeline script: scripts/ly_export.py

Two engines:
  MuseScore (default):  LilyPond → flatten → MusicXML → MuseScore CLI → MP3/WAV
  FluidSynth:           LilyPond MIDI → FluidSynth + SoundFont → WAV

Usage:
  python3 scripts/ly_export.py piece.ly                # -Score.musicxml + -Score.mp3
  python3 scripts/ly_export.py piece.ly --wav           # -Score.wav
  python3 scripts/ly_export.py piece.ly --no-mxl        # MP3 only, no MusicXML
  python3 scripts/ly_export.py piece.ly --parts         # + per-instrument audio
  python3 scripts/ly_export.py piece.ly --fluidsynth    # -Score.wav via FluidSynth+TOH
  python3 scripts/ly_export.py piece.ly --all           # everything: MusicXML + MP3 + WAV + parts + FluidSynth

Output naming follows PDF convention:
  {Name}-Score.mp3       Full score (like {Name}-Score.pdf)
  {Name}-Cello.mp3       Individual part (like {Name}-Cello.pdf)

Parts logic (--parts):
  - 1 instrument  → skipped (Score = Part, no redundant export)
  - N instruments → exports parts, but only for instruments with actual notes
                    (empty/rest-only parts are skipped via has_notes() check)

Flattening (scripts/ly_to_musicxml.py):
  - Resolves dot-notation bar variables (vc.1, upper.2, etc.)
  - Extracts time signature, key, tempo from source (no hardcoding)
  - Handles pickup/Auftakt (\partial) via {prefix}Pickup variables
  - Cleans tremolo notation (:32) not supported by python-ly
  - Removes StaffGroup/DrumStaff not supported by python-ly
